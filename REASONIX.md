# REASONIX.md — CV 静态简历网页

## Stack

- **Language:** Vanilla HTML5 / CSS3 / ES6+ (no framework, no bundler)
- **Rendering:** Data-driven — `site/main.js` reads `site/data.json`, then prefers `localStorage`
- **PDF export:** `html2pdf.js` (client-side, loaded at runtime via CDN)
- **Font subset:** Python script (`subset_font.py`) — Noto Sans SC subset via fontTools
- **Test pages:** Playwright (Python) + Puppeteer (Node.js) — standalone scripts, not a test suite

## Layout

| Path | Purpose |
|---|---|
| `site/` | Document root — deploy this directory |
| `site/index.html` | Resume page shell |
| `site/styles.css` | All styles (layout, editor, toolbar, print) — 1400+ lines |
| `site/main.js` | All JS (render, edit, paginate, zoom, import/export) — 350+ lines |
| `site/data.json` | Default resume data (JSON) |
| `site/assets/icons/` | SVG icons (phone, email, github, person) |
| `docs/` | `requirement.md` — project requirements doc |
| `example/` | Sample data + generated PDFs |
| `start.sh` / `start.bat` | Launch local HTTP server (auto-detect Python/Node.js) |

## Commands

No package.json — all commands are ad-hoc shell scripts:

- **Dev server:** `cd site && python3 -m http.server 8000` (or double-click `start.bat` / `bash start.sh`)
- **PDF test (Node):** `node test-pdf.mjs` (requires Puppeteer, expects server on port 8765)
- **PDF test (Python):** `python test-pdf-playwright.py` (requires Playwright, expects server on port 8765)
- **Font subset:** `python subset_font.py` (requires fontTools, reads `example/resume-data.json` + `site/pdfmake-export.js`)

## Conventions

- **Pure global scope** — `main.js` has no `import`/`export`; all variables and functions are global (`const`, `function`, `let` in `<script>`)
- **No build step** — edit source files directly, reload browser
- **CSS custom properties** — `:root` variables in `styles.css` for accent color, font sizes, spacing, page dimensions
- **`data-render` attributes** — HTML elements carry `data-render="profile.name"` etc.; JS resolves them against the JSON data tree
- **Editor config** — new section types only need an entry in `SECTION_CONFIG` (`main.js` top); `type` field in JSON selects the section

## Watch out for

- **`site/backup/` is gitignored** — auto-generated backups are excluded from version control; the full `data.json` path `/site/data.json` is also explicitly gitignored
- **Font files are subsets** — `NotoSansSC-*.subset.ttf` are generated by `subset_font.py` and only cover chars present in the current data + PDF export code. If you add new Chinese text, re-run the subset script or the browser may fall back to a system font
- **No package.json / no npm** — dependencies (`html2pdf.js`, `Puppeteer`, `Playwright`) are runtime-CDN or dev-only; there's no lockfile or `node_modules` in the repo
- **`site/data.json` is initial seed only** — after first edit in the browser, `localStorage` takes over. To reset: clear `localStorage` via browser DevTools or click "重置默认" in the editor
